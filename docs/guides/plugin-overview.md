# OpenBell Guard를 위한 Codex 플러그인 개요

> 문서 유형: OpenAI 공식문서 기반 프로젝트 특화 해설본  
> 대상 프로젝트: OpenBell Guard  
> 공식 원문: [Codex Plugins](https://developers.openai.com/codex/plugins)  
> 최종 확인일: 2026-06-30  
> 주의: 설치 UI, 지원 구성요소와 권한 동작은 공식 원문을 우선합니다.
> 지표·수식·오류 판정은 [단일 기준 계약](../openbell-guard-metrics-validation-contract.md)을 따릅니다.

## 1. Codex 플러그인이란 무엇입니까?

Codex 플러그인은 반복해서 사용할 업무 지식과 도구를 하나로 묶은 설치 단위입니다.

```text
플러그인
├── Skill: 어떤 절차로 일할지
├── Script: 정확한 계산과 파일 처리
├── MCP: 필요한 경우 외부 시스템 연결
├── Hook: 필요한 경우 생명주기 자동 동작
└── Asset: 아이콘, 예시와 템플릿
```

모든 요소가 필수인 것은 아닙니다. 가장 작은 플러그인은 manifest와 하나의 Skill만으로 구성할 수 있습니다.

## 2. Skill과 플러그인의 차이

- Skill은 특정 업무를 수행하는 방법입니다.
- 플러그인은 Skill과 필요한 구성요소를 설치·공유할 수 있게 포장한 단위입니다.

OpenBell Guard의 핵심은 “증권 서비스 텔레메트리 장애 분석” Skill이며, 플러그인은 이 Skill과 분석 스크립트·참고자료·합성 예시를 함께 배포합니다.

## 3. 각 구성요소가 필요한 경우

| 구성요소 | 사용 목적 | OpenBell Guard MVP |
|---|---|---|
| `plugin.json` | 플러그인 이름과 구성요소 선언 | 필수 |
| `SKILL.md` | 장애 분석 절차와 안전 경계 | 필수 |
| `scripts/` | 입력 검증, 마스킹과 통계 계산 | 사용 |
| `references/` | 서비스 경로 모델과 보고서 규칙 | 사용 |
| `assets/` | 사고·서비스 맵 예시 | 사용 |
| `.mcp.json` | 외부 관측 시스템 연결 | MVP에서는 제외 |
| `hooks/` | 이벤트 기반 자동 실행 | MVP에서는 제외 |
| `.app.json` | ChatGPT 앱·커넥터 연결 | MVP에서는 제외 |

## 4. OpenBell Guard가 해결하는 일

주요 사용자는 증권 서비스의 SRE, 백엔드 개발자와 운영 담당자입니다.

1. 장애 종료 후 합성 또는 익명화된 사고 번들 폴더 경로를 지정합니다.
2. 로컬 스크립트가 입력·시간대·지원 한도와 민감정보를 검사하고 마스킹 작업본을 만듭니다.
3. 60초 통계와 claim-evidence 관계를 `analysis.json`에 기록합니다.
4. Codex는 원본이 아니라 검증된 분석 결과로 주문·시세·관심종목·외부 연계 영향을 설명합니다.
5. 사실, 원인 가설과 판단 불가를 분리해 `openbell-report.md`를 작성합니다.
6. 검증기가 근거 참조와 비밀정보 잔존을 확인합니다.

플러그인은 실제 금융 주문이나 운영 시스템 변경을 수행하지 않습니다.

## 5. 왜 MCP 없는 Skill 중심 구조입니까?

해커톤 심사에서는 누구나 같은 입력으로 같은 핵심 결과를 확인할 수 있어야 합니다.

- 실제 카카오페이증권 모니터링 시스템에 접근할 수 없습니다.
- 외부 연결에는 인증정보와 설치 절차가 필요합니다.
- 합성 데이터로 핵심 분석 능력을 충분히 보여줄 수 있습니다.
- 로컬 스크립트는 심사자가 동작을 직접 확인하기 쉽습니다.

따라서 MVP는 Skill과 Python 스크립트로 만들고, MCP는 실제 조직 적용 단계의 선택 사항으로 남깁니다.

## 6. 설치와 권한의 의미

플러그인을 설치하면 Codex가 해당 Skill과 구성요소를 사용할 수 있게 됩니다. 설치했다고 해서 기존 승인·보안 설정이 사라지는 것은 아닙니다.

- 외부 앱이나 MCP가 있으면 별도 인증이 필요할 수 있습니다.
- 외부 시스템으로 데이터를 보내면 해당 서비스의 개인정보·데이터 정책이 적용됩니다.
- OpenBell Guard의 번들 스크립트는 로컬 읽기 전용이며 외부 API를 호출하지 않습니다.
- Codex는 원본 사고 번들을 직접 읽지 않고 `analysis.json`과 제한된 마스킹 근거만 읽습니다.
- 따라서 “Codex 서비스 전체가 네트워크와 무관하다”고 약속하지는 않습니다.
- 실제 운영 주소와 비밀정보를 플러그인 설정에 넣지 않습니다.

## 7. 플러그인, 개발 지침과 사용자 문서의 관계

```text
루트 AGENTS.md
  └─ 플러그인을 만드는 Codex의 개발 규칙

src/skills/openbell-guard/SKILL.md
  └─ 설치된 플러그인이 로그를 분석하는 실제 절차

README.md
  └─ 심사자와 사용자를 위한 설치·사용·검증 설명
```

세 문서가 같은 문제와 기능을 설명해야 하며, 구현되지 않은 기능을 문서에서 제공한다고 표현하면 안 됩니다.

## 8. 성공적인 MVP의 모습

- 별도 API 키 없이 설치하고 실행할 수 있습니다.
- 대표 사용자 요청 한 번으로 Skill이 `analysis.json`, `openbell-report.md`, `sanitization-report.md`를 재현할 수 있습니다.
- 합성 국내장 개장 피크와 외부 중개사 장애를 구분합니다.
- 원본 입력을 변경하지 않습니다.
- 모든 확인된 사실 claim에 기계 검증 가능한 evidence 참조가 있습니다.
- 근거가 부족하면 원인을 확정하지 않습니다.
- Codex 재시작 후 새 스레드에서 발견·자동 선택·명시 호출이 모두 성공합니다.
- 실제 실행한 테스트 결과가 README와 질문지에 반영됩니다.
